# POS Cloud 設計文件

## 1. 產品定位

POS Cloud 是給櫃位使用的雲端 POS 與營運管理系統。員工使用手機或平板開啟前台頁面，快速建立訂單、查看自己的班表、完成開班與下班庫存盤點；店長使用獨立後台管理報表、商品、折扣、櫃位、庫存、排班與薪資。

目前 mockup：`outputs/pos-cloud-mockup.html`

## 2. 使用者與權限

### 員工

員工登入後只看到員工工作區，不可直接切換到店長後台。

員工功能：

- 前台 POS 建立訂單
- 選擇銷售人員，支援多人同班
- 查看自己的班表
- 開班盤點與下班盤點
- 登記櫃位庫存異動：進貨、試吃、報廢、調整

員工不可操作：

- 店長報表
- 商品價格與折扣設定
- 櫃位目標設定
- 全員排班與薪資試算
- 跨櫃位庫存覆核

### 店長

店長透過獨立入口登入後進入後台。

店長功能：

- 查看營運總覽與報表
- 管理商品、禮盒、價格與折扣
- 管理櫃位與每月業績目標
- 查看跨櫃位庫存與盤點差異
- 進行每月排班
- 查看薪資試算：時薪總額 + 業績抽成總額

## 3. 前台 POS 設計

### 操作目標

前台 POS 以櫃位快速結帳為核心。員工在銷售時應能用最少步驟完成訂單。

主要流程：

1. 選商品分類：袋裝、禮盒、常用。
2. 點選商品加入訂單。
3. 若商品是可選禮盒，開啟禮盒內容選擇。
4. 選擇折扣。
5. 選擇付款方式。
6. 選擇銷售人員。
7. 完成訂單。

### 金額邏輯

前台不需要填寫實收金額。

訂單成立時：

- 銷售金額 = 商品原價小計
- 折扣後金額 = 套用折扣後的應收金額
- 應收金額 = 實收金額
- 實收金額由系統自動存入，不提供前台手動輸入

原因：

- 櫃位銷售不需要找零差額流程
- 減少員工操作欄位
- 降低錯填金額造成報表差異

### 銷售人員

銷售人員欄位為下拉選單，資料來源應為「目前櫃位當班人員」。

情境：

- 同一櫃位可能同時有多人上班
- 訂單必須歸屬到實際銷售人員
- 員工績效、每日抽成與月報表都依銷售人員計算

## 4. 商品與禮盒規則

### 袋裝商品

袋裝商品以單品計價，每個品項有自己的價格。

範例：

- 包種烏龍牛軋糖，10入/袋
- 蔓越莓牛軋糖，10入/袋
- 經典原味牛軋餅，10入/袋
- 經典原味蔥軋餅，9入/袋

### 禮盒商品

禮盒本身有價格，禮盒內口味不單獨計價。

禮盒中的每個牛軋糖口味規格為 6入/袋。

禮盒類型：

- 小禮盒：固定禮盒價格，可自行選擇 3 個口味。
- 大禮盒：固定禮盒價格，可自行選擇 8 個口味 + 1 個蔥餅。
- 發禮盒：固定禮盒價格，固定 4 個口味。
- 財禮盒：固定禮盒價格，固定 4 個口味。

前台行為：

- 小禮盒、大禮盒點選後開啟口味選擇視窗。
- 發禮盒、財禮盒直接帶入固定內容。
- 購物車顯示禮盒組成，但金額只計算禮盒價格。

後台行為：

- 店長可維護禮盒價格。
- 店長可維護可選口味清單。
- 店長可維護固定禮盒的口味組合。

## 5. 員工班表

員工端只顯示自己的班表，不顯示其他員工完整排班。

員工可看到：

- 排班日期
- 班別
- 時段
- 櫃位
- 當日提醒：開班盤點、下班盤點

員工不可修改班表。

## 6. 店長每月排班

第一版採每日兩班制。

預設班別：

- 早班：10:00-16:00，6 小時
- 晚班：16:00-22:00，6 小時

店長排班流程：

1. 選擇月份與櫃位。
2. 確認班別模板。
3. 在月曆每日早班與晚班格子指定員工。
4. 系統檢查撞班、超時、缺班、未排滿。
5. 發布班表。
6. 員工在「我的班表」查看自己的班次。
7. 月底以排班時數計算時薪總額。

薪資試算：

- 時薪總額 = 排班時數 x 員工時薪
- 抽成總額 = 每日個人業績抽成加總
- 薪資合計 = 時薪總額 + 抽成總額

## 7. 庫存管理

庫存管理會開放給櫃位員工使用，因為員工需要在上班與下班時盤點。

### 員工庫存盤點

員工可操作：

- 開班盤點
- 下班盤點
- 臨時盤點
- 進貨登記
- 試吃登記
- 報廢登記
- 調整登記

每筆庫存異動需記錄：

- 櫃位
- 品項
- 異動類型
- 數量
- 操作人員
- 時間
- 備註

### 店長庫存管理

店長可查看：

- 跨櫃位庫存彙總
- 低庫存警示
- 盤點差異
- 進貨、試吃、報廢紀錄
- 待覆核異常

店長可覆核：

- 盤點差異
- 報廢紀錄
- 異常庫存調整

## 8. 報表

店長後台報表包含：

- 每日員工業績
- 每月每位員工業績
- 總業績
- 平均客單價
- 營收金額
- 實際金流
- 業績目標達成率
- 櫃位業績統計

### 指標定義

總業績：

- 指折扣後訂單金額加總。

平均客單價：

- 總業績 / 訂單數。

實際金流：

- 依付款方式彙總的實收金額。
- 第一版中訂單應收即實收。

業績目標達成率：

- 櫃位當月總業績 / 櫃位當月目標。

## 9. 抽成規則

抽成以每日個人業績計算。

規則：

- $0 - $2,999：0%
- $3,000 - $5,000：1%
- $5,001 以上：2%

每日個人業績以訂單銷售人員歸屬計算。

## 10. 第一版範圍

第一版包含：

- 員工登入與店長登入
- 前台 POS
- 商品與禮盒規則
- 折扣設定
- 訂單與付款紀錄
- 店長報表
- 櫃位每月業績目標
- 員工我的班表
- 店長每月兩班制排班
- 薪資試算
- 員工庫存盤點
- 店長庫存管理

第一版不包含：

- 複雜混合付款
- 打卡與核准工時
- 完整進銷存會計成本
- 多層級總公司管理
- 發票串接
- 條碼掃描
